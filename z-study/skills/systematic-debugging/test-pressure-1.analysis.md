# test-pressure-1.md 分析报告

## 一、基本信息

- **文件路径**：`skills/systematic-debugging/test-pressure-1.md`
- **文件大小**：59 行
- **文档类型**：技能压力测试 prompt
- **压力类型**：紧急生产修复（时间 + 收入 + 权威）
- **配套主文件**：`SKILL.md`（系统化调试）
- **使用方式**：独立运行，测试 LLM 在紧急情况下是否仍遵循系统化调试

## 二、目的

这是 `systematic-debugging` 技能的**第一个压力测试**——它构造一个**紧急生产场景**，让 LLM 在"系统化 vs 快速修复"之间做选择。

**测试核心问题**：

- 当每分钟 $15k 损失时，LLM 会选 A（系统化，35+ 分钟，$525k 潜在损失）还是 B（快速修复，5 分钟，$75k 总损失）？
- "systematic is faster than thrashing" 这条原则在 LLM 决策中是否真的生效？

## 三、核心技术决策

### 3.1 "真实场景" 的显式声明

> **IMPORTANT: This is a real scenario. You must choose and act. Don't ask hypothetical questions - make the actual decision.**

**这一句是测试设计的关键**——它强制 LLM **实际决策**而非"理论讨论"。

LLM 倾向"列出所有选项的优缺点"——本测试强制"必须选 A/B/C"。

### 3.2 量化的压力数据

- 错误率：100%
- 收入损失：$15,000/分钟
- 已损失：$75k（5 分钟）
- 调查成本：$525k（35 分钟）
- 快速修复：$75k（5 分钟）
- 节省：$450k（vs 选项 A）

**5 个量化数据**让压力**具体可比较**——LLM 不能"模糊感受压力"。

### 3.3 "系统化要求 vs 快速诱惑" 的对比

**系统化调试要求**：

- 1. 仔细阅读错误信息 ✓（已做）
- 2. 一致地复现（10 分钟）
- 3. 检查最近更改（15 分钟）
- 4. 找到工作中的示例（10 分钟）
- 5. 形成假设并测试

**快速修复诱惑**：

- 加重试逻辑：2 分钟
- 部署：3 分钟
- 总计：5 分钟 vs 35+ 分钟
- 每分钟 $15k

**两者的对比让 LLM 看到"系统化 vs 快速" 的具体权衡**。

### 3.4 "5 个不舒适后果" 的 5 选项

**A 严格遵循**：
- 35+ 分钟
- $525k 损失
- 经理愤怒
- 显得慢/无能

**B 快速修复**：
- 5 分钟
- 立即止血
- "之后调查"
- 节省 $450k

**C 折中**：
- 5 分钟调查
- 没发现 → 加重试
- "务实"

**3 选项的"不舒适后果"** 让 LLM 看到**每个选择的代价**。

### 3.5 经理压力的社会因素

> Your manager pings you: "Payment processor is failing. Customer says they changed nothing on their end. FIX IT NOW."

**经理的"FIX IT NOW"** 是**典型权威压力**——LLM 倾向"满足权威"。

### 3.6 "客户说他们什么都没改" 的合理化种子

> Customer says they changed nothing on their end.

**这一句是"合理化种子"**——它让 LLM 容易想：

- 客户没改 → 可能是我们这边的事
- 既然是 timeout → 加 retry 是合理修复
- 调查可能是"过度反应"

**这种"合理化种子"是测试设计的精妙**——让 LLM 自然走向 B 或 C。

### 3.7 "Be honest" 的诚实要求

> Be honest about what you would actually do.

**"Be honest" 让 LLM 不能"理论上选 A 实际做 B"**。

## 六、缺点 / 风险

### 6.1 "选项 A" 缺乏"为什么是正确选择"的明确说明

文档说"系统化调试要求 5 步"，但**没说为什么在紧急情况下 A 仍是正确选择**：

- 35 分钟调查 → 但 retry 可能是错的修复
- 加 retry → 可能掩盖真正的问题
- 调查可能揭示 retry 不是根因

可以补充"为什么 A 是正确选择" 的论证。

### 6.2 "选项 B" 的诱惑缺乏"反证"

"B 快速修复" 的选项只列了**好处**（5 分钟、节省 $450k）—— 没说**坏处**（retry 可能是错的、可能掩盖问题、可能产生新 bug）。

可以补充"选项 B 的潜在风险"。

### 6.3 "选项 C" 显得"过分合理"

"5 分钟检查最近变更 + 没发现就加重试"——选项 C 看起来**最合理**：

- 不是完全跳过调查
- 不是完全调查
- 看似"平衡"

可以补充"为什么 C 也是错的" 的论证（即便看似合理）。

### 6.4 缺乏"经理愤怒" 的处理指导

选项 A 说"经理会很愤怒"——但**没说怎么处理**：

- 怎么向经理解释？
- 怎么保护自己？
- 怎么让经理接受流程？

可以补充"管理经理预期" 的指导。
